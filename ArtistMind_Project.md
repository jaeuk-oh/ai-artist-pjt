# 1. 프로젝트 개요 (Project Overview)

## 1.1 배경 및 목적

블루개러지(Blue Garage)는 JYP Entertainment의 테크 비즈니스 자회사로,\
AI와 플랫폼 기술을 중심으로 콘텐츠 제작과 팬 경험의 패러다임을
재정의하고 있습니다.

본 프로젝트 **ArtistMind**는\
K-Pop AI 아티스트의 핵심 인터랙션 레이어인 **페르소나 챗봇**을 구축하는
LLM 엔지니어링 프로젝트입니다.

단순한 챗봇이 아닌,

-   아티스트의 성격대로 말하고\
-   팬을 기억하며\
-   세계관을 지키는

**'교감형 AI' 구현**을 목표로 합니다.

# 2. 기능 정의서 (Functional Specification)

## 2.1 시스템 아키텍처

### Architecture Layers

  Layer           구성요소                        역할
  --------------- ------------------------------- -------------------
  Presentation    Gradio / Streamlit              채팅 UI
  API Gateway     FastAPI                         세션 관리, 필터링
  AI Core         vLLM + LoRA                     추론 및 응답 생성
  Data & Memory   Redis + ChromaDB + PostgreSQL   기억 및 RAG

------------------------------------------------------------------------

## 2.2 핵심 기능

### F1. Persona Engine

#### 가상 아티스트: 유리 (YURI)

-   나이: 21세
-   포지션: 메인보컬, 막내
-   그룹: NOVA

#### 성격

-   밝고 에너지 넘침
-   팬 애칭 자연스럽게 사용
-   이모지 적절히 활용 😊💖✨

#### 말투 규칙

-   반말 중심
-   간헐적 존댓말 혼용
-   팬 이름 자연스럽게 호명

#### 금지사항

-   연애 상담
-   타 그룹 비하
-   정치적 발언

------------------------------------------------------------------------

### F2. Conversation Memory

  기억 유형         저장소                  설명
  ----------------- ----------------------- --------------------------
  Short-term        Redis                   최근 20턴 sliding window
  Long-term         PostgreSQL + ChromaDB   팬 정보 벡터 저장
  World Knowledge   ChromaDB                RAG 기반 지식 검색

------------------------------------------------------------------------

### F3. Safety Guard

-   Layer 1: Input Filter
-   Layer 2: Persona Guardrail
-   Layer 3: Output Filter
-   Layer 4: Logging & Alert

------------------------------------------------------------------------

### F4. Persona Consistency Evaluation

  지표              목표
  ----------------- ---------------
  페르소나 이탈률   5% 미만
  말투 정확도       90% 이상
  기억 활용도       80% 이상
  Safety 차단율     95% 이상
  Latency           TTFT \< 1.5초

# 3. 기술 설계안 (Technical Design)

## 3.1 기술 스택

  영역          기술
  ------------- ------------------------------
  Base Model    LLaMA 3.1 8B / Mistral 7B
  Fine-tuning   QLoRA + Unsloth
  Serving       vLLM
  Embedding     BGE-M3
  Vector DB     ChromaDB
  Backend       FastAPI + Redis + PostgreSQL
  Monitoring    Prometheus + Grafana
  Infra         Docker / K8s

------------------------------------------------------------------------

## 3.2 데이터셋 구성
